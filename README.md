# sporthub-ayx-guide

这是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅用于存放各类静态 HTML 文档，供社区参考和查阅。

## 项目简介

本仓库主要收集、整理并托管一系列独立的 HTML 页面。这些页面可能是工具、说明文档、演示页面或其他形式的静态内容。所有内容均以 HTML 文件形式直接存放于仓库中，无需构建或编译即可通过 GitHub Pages 等静态托管服务访问。

## 目录说明

```
├── README.md          # 本文件
├── index.html         # 可选：根目录入口页面
├── pages/             # 存放主要 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/            # 存放页面所需的静态资源（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── docs/              # 可选：存放额外文档或说明
```

- **pages/**: 所有独立 HTML 页面存放于此，每个文件为一个完整页面。
- **assets/**: 页面引用的资源文件，按类型分目录存放。
- **docs/**: 其他辅助文档，如使用说明、变更日志等。

## 页面归档说明

- 每个 HTML 页面均为独立文件，可单独访问，不依赖其他页面。
- 页面内容可能随时间更新或替换，历史版本保留在 Git 提交记录中。
- 页面名称应尽量反映其内容或用途，避免使用无意义命名。
- 若页面需要引用外部资源（如 CDN 库），请确保其长期可用或一并归档至 assets 目录。

## 维护说明

- 本仓库由社区维护，欢迎提交 Issue 或 Pull Request 来添加新页面或修正现有内容。
- 提交新页面时，请确保：
  - 页面为独立 HTML 文件，不包含外部依赖（或依赖已归档）。
  - 文件名使用小写字母、数字和连字符，避免特殊字符。
  - 页面内容不违反 GitHub 服务条款及适用法律法规。
- 对于不再需要的页面，建议在提交删除前确认无外部引用。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/sporthub-ayx-guide.git
   ```
2. 直接在浏览器中打开 pages/ 目录下的 HTML 文件即可预览。
3. 若启用 GitHub Pages，可将 pages/ 目录设为发布源，使页面可通过 URL 直接访问。
